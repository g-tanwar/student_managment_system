===========================================
SYSTEM ARCHITECTURE — Student Management System (EduPortal)
===========================================

TECH STACK:
- Frontend  : React.js, React Router, Axios, Lucide Icons
- Backend   : Node.js + Express.js
- Database  : MongoDB (Mongoose ODM)
- Auth      : JWT (JSON Web Tokens)
- Validation: Joi
- Hosting   : Render (Backend), Vite Dev Server (Frontend)

===========================================

===========================================
MVC + SERVICE LAYER PATTERN:
===========================================

Request
  --> Route
    --> Middleware (auth + validate)
      --> Controller (handle req/res)
        --> Service (business logic)
          --> Model (mongoose)
            --> MongoDB
          <-- Model
        <-- Service
      <-- Controller
  <-- Response

| Teacher CRUD       |  YES  |   NO    |   NO    |

===========================================
FRONTEND ROUTING:
===========================================

Public Routes:
- /login --> LoginPage

Protected Routes (require JWT):
- /dashboard  --> StudentDashboard
- /attendance --> MyAttendance
- /fees       --> FeeStatus
- /notes      --> Notes
- /pomodoro   --> Pomodoro
- /schedule   --> Schedule
- /goals      --> Goals
- /profile    --> Profile

Auth Flow:
- No token --> redirect /login
- Valid token --> redirect /dashboard
- 401 response --> clear token, redirect /login
