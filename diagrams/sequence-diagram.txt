===========================================
SEQUENCE DIAGRAM 4 — Record Fee Payment
===========================================

Admin          Frontend        Backend         Database
  |                |               |               |
  |  Enter Payment |               |               |
  |  Details       |               |               |
  |--------------->|               |               |
  |                | POST /api/v1  |               |
  |                | /fees/:id/pay |               |
  |                | +JWT Token    |               |
  |                |-------------->|               |
  |                |               | protect()     |
  |                |               | authorize(ADMIN)|
  |                |               | findById(fee) |
  |                |               |-------------->|
  |                |               | Fee Object    |
  |                |               |<--------------|
  |                |               | Check status  |
  |                |               | != PAID       |
  |                |               | Validate amount|
  |                |               | Update paidAmount|
  |                |               | Push payment  |
  |                |               | history       |
  |                |               | fee.save()    |
  |                |               |-------------->|
  |                |               | Updated Fee   |
  |                |               |<--------------|
  |                |  200 OK       |               |
  |                |<--------------|               |
  |  Payment Logged|               |               |
  |<---------------|               |               |
